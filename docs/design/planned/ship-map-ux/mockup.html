1 · In-transit marker — which glyph style?

Style A

Arrowhead

Filled chevron pointing at the destination + soft glow. Minimal, unmistakably directional, reads at any zoom.

Style B

Directional pill

Solid fleet-colour pill with a pointed leading edge + dark ship glyph. Matches the docked pill family; biggest, most "labelled".

Style C

Comet / trail

Glowing head with a tapering motion trail behind it. Strong "in-motion" feel; direction read from the trail, not a point.

2 · Docked pill (sits on the system glyph) — replaces the faint pulse ring

Docked

Ship icon + count

Solid pill anchored top-right of the glyph. Same fleet colour as the in-transit marker → one visual language for "your fleet".

Before → after

Why replace the ring

Today: a faint cyan pulse ring (alpha 0.4→0) + tiny yellow "1 SHIP" text — lost against glow, gateway dots, fuel labels. The pill is solid, high-contrast, scannable.

3 · Progressive disclosure — markers always on, lines on demand

Default

Marker only

You always see a ship is out there and where — no line, no clutter.

Hover

Ghost route + ETA

Transient: dashed route + a small destination/ETA tooltip. Nothing persists.

Click

Select on map

Solid animated route + compact ship card. Map stays visible — a map-native ship selection.

4 · Clustering — many ships, no spaghetti

Anti-clutter

Spread by progress · cluster with count

Separate ships sit at different points along the lane (different departure ticks). Convoys = one marker. Overlapping solos collapse to one glyph with a count badge until they separate.

Route lines are reconstructed client-side with the existing shortest-path finder (origin → destination), animated in the travel direction. ETA = arrivalTick − currentTick. A "Routes" overlay toggle (next to Trade Flows / Price) draws every in-transit route at once for the overview.
